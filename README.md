# RAG Knowledge Assistant

A tiny, opinionated, open-source **RAG knowledge assistant** built for learning and experimentation.
It exposes a FastAPI service that can ingest documents into an in-memory knowledge base and answer
queries with relevant chunks.

This project focuses on a clean retrieval-first flow (no LLM generation step yet).
It is intentionally small, dependency-light, and easy to extend.

---

## Features

- Document ingestion via HTTP API
- Overlapping chunking for better context preservation
- TF-IDF / cosine-similarity retrieval over indexed chunks
- Optional semantic retrieval with sentence-transformer embeddings (extra install)
- Configurable retrieval settings per request
- Document deletion and clear/reset endpoints
- Health and statistics endpoints
- OpenAPI docs auto-generated by FastAPI
- Unit tests and CI workflow

---

## Project structure

- `src/rag_assistant/api.py` — FastAPI app and HTTP endpoints
- `src/rag_assistant/knowledge_base.py` — retrieval engine and in-memory store
- `src/rag_assistant/models.py` — request/response schemas
- `src/rag_assistant/cli.py` — convenience CLI launcher
- `tests/` — starter test suite
- `.github/workflows/ci.yml` — CI pipeline

## Quickstart

### 1) Install dependencies

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
```

For semantic retrieval:

```bash
pip install -e ".[dev,embeddings]"
```

### 2) Run API

```bash
uvicorn rag_assistant.api:app --reload --app-dir src
```

### 3) Ingest content

```bash
curl -X POST http://localhost:8000/ingest \
  -H "Content-Type: application/json" \
  -d '{"source_id":"guide-1","content":"RAG combines retrieval with generation."}'
```

### 4) Query

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question":"What is RAG?","top_k":3}'
```

### Semantic query example

```bash
curl -X POST http://localhost:8000/query/semantic \
  -H "Content-Type: application/json" \
  -d '{"question":"What is RAG?","top_k":3,"embedding_provider":"sentence_transformers","embedding_model":"sentence-transformers/all-MiniLM-L6-v2"}'
```

Local TF-IDF alternative:

```bash
curl -X POST http://localhost:8000/query/semantic \
  -H "Content-Type: application/json" \
  -d '{"question":"How can embeddings be generated?","top_k":3,"embedding_provider":"local_tfidf","local_dimensions":64}'
```

### 5) Optional: run tests

```bash
pytest
```

## API reference

- `GET /health` - service health
- `GET /stats` - corpus/index stats
- `POST /ingest` - ingest one document
- `POST /ingest/bulk` - ingest many documents
- `DELETE /documents/{source_id}` - remove all chunks for a source
- `POST /query` - retrieve top-k relevant chunks
  - `retrieval`: `"tfidf"` (default) or `"semantic"`
  - `embedding_model` (optional): model override for semantic retrieval
- `POST /query/semantic` - semantic-only query endpoint
  - `embedding_provider`: `"sentence_transformers"` (default), `"local"` / `"local_hash"`, or `"local_tfidf"`
  - `local_dimensions`: optional local embedding dimensions (8-4096)
- `DELETE /clear` - clear entire corpus

Docs UI: `http://localhost:8000/docs`
